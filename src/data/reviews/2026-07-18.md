## 智能物流 Agent 系统

### 纠错发布的选择权与验收边界被重新落到会话流程

纠错会话修复明确了人工标准发布、人工风险发布、完整 Agent Improve 和暂不发布四类路径，并完成了对应提交。流程不再依赖模型自行猜测用户是否允许发布。

**重要性。** 发布决策从隐含推断变成显式状态，降低了用户尚未授权时误发布，或发布后跳过验收的风险。

### 纠错发布主链路完成真实场景回归

针对多类目标补丁、运行时资产隔离、历史残留清理和发布前阻断新增了隔离场景测试，并通过浏览器闭环验证了摘要、确认、发布和规则测试结果。

**重要性。** 关键流程从单点修补推进到端到端验收，后续再改纠错链路时更容易发现状态泄漏和错误放行。

### Trajectory 评估从直觉判断转为真实样本校验

真实样本分析显示，过程轨迹的高置信判断仍会把地址规范化、转换规则和漏拆车等错误误判为正确，因此不能直接作为自动放行标准。

**重要性。** 这把质量门禁从“过程看起来合理”拉回到业务结果可验证，避免把解释链路误当成事实正确性。

### 显式卸货地址不再被通用回退逻辑误处理

新增了显式卸货标签的正向保护测试，确保确定性规则直接提取地址且不进入 AI 回退路径；相关测试和构建均通过，并形成独立提交。

**重要性。** 明确输入可以继续由稳定规则处理，避免模型回退带来的额外成本和语义漂移。

## Agent Blog

### Agent Blog 的产品介绍与个人博客展示完成解耦

博客首页改为可配置介绍内容，修正了产品宣传文案直接占据个人博客首页的问题，并完成构建、测试和拉取请求。

**重要性。** 同一套主题系统可以服务不同博客身份，不再把产品官网叙事强加给每个站点。

## 企业管理平台

### 本地开发默认常驻任务从二十余个降至两个

开发脚本改为预构建共享依赖并复用缓存，默认只常驻核心前后端进程，同时保留全量 watcher 模式。

**重要性。** 大型多模块仓库的日常启动负担显著降低，同时没有牺牲修改共享包时的完整开发模式。

## 开发工具链

### 项目级 Skill 与全局安装完成去重

按全局版本优先的规则移除了项目内重复 Skill，并验证所有全局 fallback 均存在，项目目录只保留自身独有能力。

**重要性。** 减少了同名 Skill 的行为分叉和维护成本，也让项目实际依赖来源更清晰。
