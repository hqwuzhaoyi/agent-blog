## 智能物流 Agent 系统

### 纠错对话与自动发布风险被系统化收敛

多轮评审识别出旧验证证据复用、发布并发覆盖、发布后验收失败仍关闭会话等关键风险，并将改进周期、发布围栏、补偿验证和状态投影整理为独立规格与实施任务。另一路修复报告显示，纠错发布主流程及展示状态已通过大规模回归验证。

**重要性。** 风险从零散代码问题转化为可实施的状态机和验收边界，降低了未经重新验收或并发覆盖后错误发布的可能性。

### 显式卸货地址不再被通用回退逻辑误处理

新增了显式卸货标签的正向保护测试，确保确定性规则直接提取地址且不进入 AI 回退路径；相关测试和 TypeScript 构建均通过，并已形成独立提交。

**重要性。** 明确输入可以继续由稳定规则处理，避免模型回退带来的额外成本和语义漂移。

### 项目级 Skill 与全局安装完成去重

按全局版本优先的规则移除了七个项目内重复 Skill，更新锁定信息并验证所有全局 fallback 均存在，项目目录只保留自身独有能力。

**重要性。** 减少了同名 Skill 的行为分叉和维护成本，也让项目实际依赖来源更清晰。

## Agent Blog

### Agent Blog 的产品介绍与个人博客展示完成解耦

博客首页改为可配置介绍内容，修正了产品宣传文案直接占据个人博客首页的问题，并完成构建、测试、提交和拉取请求。

**重要性。** 同一套主题系统可以服务不同博客身份，不再把产品官网叙事强加给每个站点。

## 开发工具链

### 遥测支持从干净主线重新构建

遥测改动从主线基线逐文件重做，排除了此前混入的财务流程、自动归档和其他无关逻辑；发布钩子、兼容运行时和多组 schema 校验全部通过。

**重要性。** 交付范围恢复到最小必要集合，减少了无关功能随遥测升级进入发布版本的风险。

## 企业管理平台

### 本地开发默认常驻任务从二十余个降至两个

开发脚本改为预构建共享依赖并复用缓存，默认只常驻核心前后端进程，同时保留全量 watcher 模式；报告显示二次启动可命中缓存，干净环境也能正常启动。

**重要性。** 大型多模块仓库的日常启动负担显著降低，同时没有牺牲修改共享包时的完整开发模式。
